# General Development Rules

- **Browser Verification**: When implementing a big change, always check the app in a browser to make sure it works as expected.
- **Research**: When you are unsure about something, always use docfork to search for information.
- **Security**: Always check for security vulnerabilities when implementing something and make sure you put measures in place to combat those potential security issues.
- **MCP Usage**: Always use docfork when unsure about a package or you need some information about implementing a specific feature.
- **Code Reviews**: Always have a code review before merging any changes.
- **Documentation**: Always update the documentation when making changes to the codebase.
- **Commit Messages**: Always use clear and descriptive commit messages.
- **Code Formatting**: Always use Prettier for code formatting.
- **TypeScript**: Always use TypeScript for type safety.
- **Linting**: Always use ESLint for linting.
- **Testing**: Always write tests for your code.
- **Code Formatting**: Always format your code using Prettier.
